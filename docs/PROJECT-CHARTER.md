# Project Charter

Project: EduKanban.

Purpose: reduce task chaos in student group chats by giving students, group leads and teachers one transparent board with deadlines, roles, schedule-aware planning and progress analytics.

Key elements:

1. Product goal: working MVP for an educational group.
2. Problem: lost tasks, unclear ownership, deadline risks.
3. Target users: student, starosta, teacher, project manager, team lead, admin.
4. Scope: auth, RBAC, Kanban, comments, analytics, schedule import, PERT AI, local planning.
5. Out of scope: external LMS integration and real LLM provider in this iteration.
6. Success metrics: active users, overdue tasks, on-time completion, average close time, schedule load, satisfaction.
7. Constraints: local launch, PostgreSQL, REST API, demo data.
8. Risks: incomplete schedule, wrong estimates, role misconfiguration, demo environment failure.
9. Stakeholders: project team, teacher, student group.
10. Deliverables: source code, README, database schema, seed data, tests, docs, demo scenario.
11. Timeline: MVP-first iterative delivery.
12. Acceptance: project builds, tests pass, demo accounts work, main scenarios are visible in UI.
