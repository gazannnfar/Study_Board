# EduKanban
### Учебная канбан-система с ролями, расписанием, PERT и AI-помощником

MVP для учебных групп: задачи, дедлайны, аналитика и планирование по реальному расписанию.

Команда: _____________  •  Дата защиты: _____________

---

## 1. Проблема

- Задачи учебной группы теряются в чатах и переписке.
- Непонятно, **кто отвечает** за задачу и **когда** дедлайн.
- Нагрузка по парам не учитывается — планируют «на глаз».
- Преподавателю не видно прогресс и качество работы.

> Нужна одна прозрачная доска: роли, сроки, расписание, аналитика.

---

## 2. Пользователи и роли (RBAC)

| Роль | Что делает |
| --- | --- |
| **STUDENT** | свои задачи, смена статуса |
| **STAROSTA** | распределение задач, импорт расписания |
| **TEACHER** | прогресс, комментарии, оценки |
| **PROJECT_MANAGER** | задачи проектов между группами |
| **TEAM_LEAD** | планирование задач команды в слоты |
| **ADMIN** | пользователи, роли, активность |

Права централизованы в модуле `rbac` и покрыты тестами.

---

## 3. Архитектура

```text
React SPA (TS, Vite, Zustand, dnd-kit)
        │  REST + JWT
Express API (TypeScript, модульная структура)
  ├─ auth / rbac          ── JWT + права
  ├─ tasks / groups       ── канбан, проекты
  ├─ schedule             ── импорт расписания (JSON/CSV)
  ├─ planning             ── свободные окна + рекомендации
  ├─ ai                   ── PERT + 2 режима (STRICT/LIGHT)
  └─ analytics            ── KPI
        │  Prisma ORM
PostgreSQL
```

Каждый модуль: `routes` → `service` → `schemas` (Zod). Ошибки единообразны: `{ error, message }`.

## 4. Демо: доска и роли

- Логин `teacher@edukanban.local` / `password123`.
- Канбан-доска: BACKLOG → TODO → IN_PROGRESS → REVIEW → DONE.
- Drag-and-drop меняет статус (dnd-kit).
- Комментарии и выставление оценки преподавателем.
- Админ-панель: управление ролями и активностью пользователей.

---

## 5. Демо: расписание и локальное планирование

- **Импорт расписания** (CSV/JSON): пары, время, преподаватель, аудитория, тип, тема.
- Если расписание на картинке → OCR/ручной перенос в CSV-шаблон → импорт.
- Планировщик строит **занятые интервалы** (пары + запланированные задачи).
- `/planning/free-slots` → свободные окна; `/task-recommendations` → какие задачи и когда делать.
- Привязка задачи к слоту: `/planning/assign-slot`.

---

## 6. Демо: AI-помощник + PERT (два режима)

**PERT:** `expected = (O + 4·M + P) / 6`, плюс разброс и уверенность.

На одной задаче два режима:

- **AI_STRICT** — формально: O/M/P/E, риски, зависимости, 5 подзадач, осторожная оценка по слоту.
- **AI_LIGHT** — кратко: ожидаемое время + ближайшее действие, 2–3 пункта.

AI = `RuleBasedAiAssistant` за интерфейсом `AiAssistantService` — заменяется реальным LLM без правок фронтенда.

## 7. Качество и результаты

- **Аналитика (KPI):** % в срок, активные пользователи, среднее время закрытия, просрочки, распределение по статусам, удовлетворённость, нагрузка по расписанию, PERT-сводка.
- **Тесты:** 8 unit-тестов (RBAC, AI strict/light, PERT) — зелёные.
- **Статический анализ:** ESLint; **типобезопасность:** `tsc --noEmit`.
- **CI/CD:** GitHub Actions — install → lint → typecheck → test → build.
- **Запуск за 1 шаг:** `npm run dev`. Деплой: Vercel + Railway/Render + managed PostgreSQL.
